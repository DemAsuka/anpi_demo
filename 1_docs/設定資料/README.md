## 1. 概要

このフォルダは、要件定義とは別で「DBの初期値」「運用設定のたたき台」「対応表」など、**設定データ（データベース化しやすい資料）**をバージョン管理するための置き場です。

要件定義書は「何を実現するか（要求/方針）」、本フォルダは「どう設定するか（初期値/ドラフト/運用の具体）」を扱います。

## 2. 置くもの（例）

- 発動条件メニュー（トリガー）対応表
- Vercel用の監視設定（JSON）
- Slack投稿テンプレ（種別ごとの初期文面）
- 対象地域（拠点/県/市区町村）リスト（初期値）
- 発動条件（閾値/クールダウン等）の初期値

## 3. 運用ルール（推奨）

- ドラフト資料はファイル名に `_ドラフト` を付ける
- 変更時は、可能なら「なぜ変えたか」をファイル冒頭の更新履歴に追記する
- 実装でDBに反映したら、該当箇所に「実装反映済み（いつ/どこ）」をメモする

## 4. ファイル一覧

- `発動条件メニュー対応表_ドラフト.md`：要件定義ベースの人間向け対応表（ドラフト）
- `JMA_SOURCES_JSON_v0.1.json`：上記対応表をVercel実装向けに変換したJSON（フィードURL/ルーティング/更新検知方針）
