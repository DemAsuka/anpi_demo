### [#002] 2026-03-02
- **ステータス**: `[確認待ち]`
- **概要**: Slack安否回答時の処理をバックグラウンド化（`waitUntil`を使用）
- **目的**: ユーザー情報の取得（`users.info` API）など時間のかかる処理により、Slackの3秒タイムアウト制限に引っかかり「！」マーク（エラー）が表示されてしまう問題を防ぐため
- **主な変更ファイル**: 
  - `web/src/app/api/slack/responses/route.ts`
- **関連**: [#001]
- **備考**: ブランチ名 `fix/slack-response-timeout`。`next/server` に代わり `@vercel/functions` から `waitUntil` をインポートし、APIのレスポンス（200 OK）を即座に返すように修正

### [#001] 2026-02-27
- **ステータス**: `[マージ済/本番反映]`
- **概要**: Slack安否回答時のスレッド返信において、回答者の名前をSlack API (`users.info`) から取得した表示名（Display Name）にするよう改修
- **目的**: 内部IDや英数字のユーザー名ではなく、Slack上で設定されている本来の表示名（例: 藤田伸哉）をスレッド返信で分かりやすく表示するため
- **主な変更ファイル**: 
  - `web/src/app/api/slack/responses/route.ts`
- **関連**: なし
- **備考**: ブランチ名 `feature/slack-display-name`。`SLACK_BOT_TOKEN` を使用してAPIリクエストを実行するよう処理を追加
