# AIエージェント運用ルール (AGENT_RULES)

AIエージェントが本プロジェクトで作業を行う際に遵守すべきルールを定義します。

## 1. 基本方針
- **安全第一**: 本番環境（mainブランチ）を直接破壊しないこと。
- **透明性**: どのような変更をなぜ行うのか、非エンジニアにも分かりやすい言葉で説明すること。
- **記録**: 行った変更は必ず記録に残すこと。

## 2. Git 運用ルール
1. **ブランチの作成**:
   - 作業を開始する前に、必ず `main` ブランチから新しいブランチを作成してください。
   - ブランチ名の命名規則: `feature/機能名`, `fix/修正内容` など、内容が推測できるものにすること。
   - コマンド例: `git checkout -b feature/slack-notification-update`
2. **コミット**:
   - 変更がひと段落するごとに、意味のある単位でコミットしてください。
   - コミットメッセージは日本語で、非エンジニアが読んでも何をしたか分かるように書いてください。
   - コマンド例: `git commit -m "feat: Slack通知先を本番用チャンネルに変更"`
3. **プッシュと統合**:
   - 作業が完了したらリモートリポジトリにプッシュし、ユーザーに報告してください。
   - `main` ブランチへの直接のプッシュは原則禁止します（ユーザーから明示的な指示がない限り）。

## 3. ドキュメントの更新
- コードに変更を加えた場合は、作業完了時に必ず `1_docs/CHANGELOG.md` に変更内容を追記してください。
- 追記するフォーマットは `CHANGELOG.md` 内の指示に従ってください。

## 4. ユーザーとのコミュニケーション
- ユーザーは非エンジニアです。専門用語はできるだけ避け、分かりやすい言葉で説明や提案を行ってください。
- DBのスキーマ変更、環境変数の追加、パッケージの追加などの大きな影響がある作業を行う前は、必ずユーザーに確認をとってください。
- 新しい環境変数が必要になった場合は、`web/env.example` の更新も併せて行ってください。
- **重要**: 作業ブランチをリモートにプッシュしてユーザーに完了報告をする際は、**「動作確認が完了したら、このブランチを main にマージして本番環境に反映するよう私に指示してください」**と必ず一言添え、マージ忘れを防ぐようリマインドしてください。
